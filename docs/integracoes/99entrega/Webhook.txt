Webhook Notification Integration Document: Uses HMAC-SHA256 Signature Verification.

This document will guide you through the process of securely integrating our webhook notification service and using HMAC-SHA256 for signature verification to ensure data security and integrity. Follow the steps below to configure your system to receive and verify webhook notifications from our servers.

Note: We cannot guarantee 100% delivery of webhook notifications due to complex network issues. Do not rely solely on webhook notifications in your business logic; instead, use the order details API to ensure you have the latest information.

1. Prerequisites
You should have basic programming knowledge and be familiar with handling HTTP requests.
Ensure your system can receive HTTPS POST requests.
Prepare a publicly accessible URL to receive webhook notifications.
2. Key Generation and Management
2.1 View Key
Open the 99Entrega website and navigate to the developers page, then locate the "Webhooks" setting to view your webhook key. This key will be used to calculate the signature, ensuring data has not been tampered with.

2.2 Key Security
Confidentiality: Keep your key secure and do not disclose it to any third party.

3. Webhook Notification Format
Each time an event is triggered, we will send a POST request to your provided webhook URL containing the following:

Headers:

Content-Type: application/json
X-Webhook-Signature: HMAC-SHA256 signature value
Body:

JSON formatted data. Specific content varies by event type.
4. Signature Verification Process
To verify that the received webhook notification is from us and has not been tampered with, follow these steps to calculate and verify the signature:

4.1 Calculate Signature
Extract the required information: Get the X-Webhook-Signature from the request header.
Construct the signature string: Use the HTTP body content as the signature string.
Use the HMAC-SHA256 algorithm: Use your webhook key to encrypt the string from step 2 with HMAC-SHA256.
Base64 encoding: Convert the resulting binary signature to Base64 encoding format.
4.2 Verify Signature
Compare signatures: Compare your calculated signature with the X-Webhook-Signature value in the request header.

If the signatures match, the webhook notification is considered trustworthy.

5. Sample Code (Python)
Here is a simple example in Python demonstrating how to verify the webhook notification signature:

python
Python
import hmac
import hashlib
import base64
import time
from flask import Flask, request

app = Flask(__name__)

# Replace with your webhook key
YOUR_SECRET_KEY = 'your_webhook_secret_key_here'

@app.route('/your-webhook-endpoint', methods=['POST'])
def webhook_receiver():
    # Extract signature from request header
    received_signature = request.headers.get('X-Webhook-Signature')


    # Construct signature string
    signature_string = f"{request.data.decode('utf-8')}"

    # Use your key to calculate the signature
    calculated_signature = hmac.new(YOUR_SECRET_KEY.encode('utf-8'), signature_string.encode('utf-8'), hashlib.sha256).hexdigest()

    # Verify signature
    if calculated_signature != received_signature:
        return 'Signature verification failed', 401

    # If signature passes verification, process webhook content...
    process_webhook_data(request.json)
    return 'Webhook received and processed', 200

def process_webhook_data(data):
    # Handle the received webhook data here
    print("Received data:", data)

if __name__ == '__main__':
    app.run(debug=True)

Webhook-Events
1. Request Body Format
The format of the webhook request body is fixed as follows:

json
{
  "event": "{EventType}",
  "event_id": "{EventId}",
  "message": "{Message Body}",
  "timestamp": 1730114201
}

Field meanings are as follows:

event: The type of event for the notification. Different event types correspond to different message content and formats, which will be listed out.
event_id: The unique ID for the event notification. Note that the same event may be requested multiple times in the event of network issues. You can use the event_id to determine whether the request has been received.
message: The specific content of the webhook notification, formatted as a JSON string. Different event types correspond to different message content.
timestamp: The time the event occurred.
